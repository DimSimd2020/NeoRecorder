# План NeoRecorder: путь к свержению OBS с позиции №1

Этот документ фиксирует амбициозный, но практический план развития NeoRecorder из сильного рекордера в современную студийную платформу нового поколения.

Цель документа:

* задать единый вектор разработки;
* не распыляться на случайные фичи;
* не превращать проект в хаотичную свалку возможностей;
* строить не просто клон OBS, а продукт, который превосходит его по удобству, скорости работы и интеллектуальности.

---

# Главная цель

NeoRecorder должен стать не просто альтернативой OBS.

NeoRecorder должен стать более современным классом продукта:

**студийная система записи, стриминга, сценической композиции, быстрой пост-обработки и AI-помощи внутри одного приложения.**

Не просто:

* запись экрана;
* набор сцен;
* стриминг.

А полный цикл:

* собрать сцену;
* записать или провести эфир;
* автоматически разобрать результат;
* быстро нарезать контент;
* экспортировать готовые материалы.

---

# Принцип позиционирования

Мы не делаем «ещё один OBS».

Мы делаем:

**OBS + Resolve Lite + AI production copilot**

Это означает три продукта в одном:

## 1. Live Studio

Функции прямой записи и стриминга:

* сцены;
* источники;
* preview/program режим;
* переходы;
* микшер;
* hotkeys;
* browser source;
* overlays;
* запись;
* стриминг.

## 2. Post Production Lite

Быстрая пост-обработка без тяжёлой профессиональной монтажки:

* таймлайн;
* клипы;
* обрезка;
* базовые переходы;
* работа с субтитрами;
* экспорт под разные форматы.

## 3. AI Layer

Интеллектуальная помощь после записи и в процессе настройки:

* транскрипция;
* генерация субтитров;
* поиск лучших моментов;
* автоматическое выделение клипов;
* генерация заголовков и описаний;
* подсказки по настройке сцены;
* диагностика проблем студии и записи.

---

# Почему именно так можно победить OBS

Лобовая война с OBS только по количеству функций — плохая стратегия.

OBS силён:

* зрелостью;
* плагинами;
* привычкой пользователей;
* огромной историей развития.

Победа возможна только если дать пользователю не «то же самое, но чуть иначе», а **более выгодный workflow**.

Пользователь должен ощущать:

* быстрее старт;
* проще настройка;
* меньше ручной возни;
* меньше переключений между программами;
* больше автоматизации;
* лучшее восстановление после ошибок;
* более понятная архитектура сцен и источников.

Именно это и есть реальный путь к тому, чтобы откусить у OBS лидерство.

---

# Что уже является сильной базой

На текущем этапе у проекта уже есть фундамент, который нельзя разрушить и который нужно усиливать:

* scene-based backend;
* layered composition model;
* preview/program workflow;
* transitions;
* multi-monitor support;
* encoder fallback;
* automated test suite;
* studio-oriented architecture.

Это уже не прототип игрушки.
Это начало студийного ядра.

---

# Что нельзя делать

Во имя скорости нельзя ломать архитектуру.

Запрещено:

* пихать бизнес-логику в GUI;
* городить фичи без явной модели состояния;
* смешивать realtime-pipeline и post-processing без границ;
* добавлять AI в критический live-path, если это создаёт лаги;
* писать «временные решения», которые потом останутся навсегда;
* позволять агентам менять всё подряд без чёткой зоны ответственности.

Если проект станет хаотичным, он проиграет OBS ещё до релиза.

---

# Стратегический план

Развитие должно идти в пять больших этапов.

---

# Этап 1. Абсолютно стабильное студийное ядро

Цель этапа:

сделать ядро настолько надёжным, чтобы на него можно было навешивать любые фичи без постоянного страха поломки.

## Нужно закончить

* явная state machine для записи;
* явная state machine для studio session;
* жёсткая модель владения ресурсами;
* нормальная обработка device lost / encoder fail / window gone / monitor gone;
* единый механизм fallback с обязательным логированием;
* телеметрия runtime-ошибок;
* стабильное завершение всех фоновых процессов;
* строгая изоляция domain/application/infrastructure/UI.

## Результат этапа

После этого NeoRecorder должен быть предсказуемым приложением, которое не разваливается от типовых desktop-сценариев.

---

# Этап 2. Полноценная студийная модель уровня modern OBS

Цель этапа:

сделать NeoRecorder не просто рекордером, а настоящей live studio system.

## Нужно реализовать

* полноценные transforms источников:

  * position;
  * scale;
  * crop;
  * rotation;
  * visibility;
  * lock.
* scene nesting;
* source duplication;
* scene duplication;
* source locking/hiding;
* browser source;
* image source;
* video/media source;
* text source;
* color source;
* audio input/output sources;
* улучшенный audio mixer;
* peak/RMS meters;
* mute/solo;
* monitoring;
* gain and sync offset.

## Результат этапа

На выходе пользователь должен иметь всё основное, что нужно для живой студийной работы без необходимости сразу идти в OBS.

---

# Этап 3. Движок, который делает всё быстрее и чище

Цель этапа:

сделать внутренний pipeline сильнее и современнее, чем просто набор ffmpeg-команд.

## Нужно реализовать

* realtime compositor с понятным render graph;
* подготовку к GPU-render path;
* нормальный preview renderer;
* предсказуемое frame pacing;
* раздельные очереди для render / audio / encode;
* контроль dropped frames;
* контроль encode lag;
* zero-copy там, где это возможно;
* более мощный filters pipeline.

## Результат этапа

NeoRecorder должен ощущаться быстрым, стабильным и современным, а не как GUI-обёртка над ffmpeg.

---

# Этап 4. Resolve Lite внутри NeoRecorder

Цель этапа:

убрать лишний разрыв между записью и дальнейшей обработкой контента.

## Нужно реализовать

* media bin;
* список записей внутри приложения;
* быстрый просмотр записанного материала;
* выделение фрагментов;
* clip trim;
* split;
* базовый timeline;
* chapter markers;
* export presets;
* экспорт в 16:9 / 9:16 / 1:1;
* встроенные субтитры;
* burn-in captions toggle.

## Результат этапа

Пользователь может не только записать материал, но и сразу превратить его в готовый контент без перехода в тяжёлую монтажку.

---

# Этап 5. AI-платформа поверх студии

Цель этапа:

сделать NeoRecorder не просто инструментом записи, а интеллектуальным помощником создателя контента.

## Нужно реализовать

* транскрипция после записи;
* автоматическая генерация субтитров;
* разбиение записи на главы;
* выявление лучших моментов;
* предложения по short-form клипам;
* автоматические названия и описания;
* рекомендации по настройке битрейта, кодека и сцены;
* объяснение сбоев простым языком;
* диагностика проблем микрофона, сцены и источников;
* AI-помощник для сборки сцены.

## Жёсткое правило

AI не должен ломать live-часть.

Сначала AI работает:

* до записи как помощник настройки;
* после записи как анализатор материала.

В критический realtime-path AI добавляется только там, где гарантированно не рушится производительность.

---

# Минимальный набор возможностей для реального удара по OBS

Чтобы рынок вообще начал воспринимать NeoRecorder как серьёзную альтернативу OBS, минимум должен включать:

* сцены и источники;
* preview/program режим;
* transitions;
* browser source;
* audio mixer;
* streaming outputs;
* стабильный encoder fallback;
* profiles/presets;
* scene collections;
* recovery after errors;
* понятную diagnostics system;
* встроенную быструю пост-обработку;
* AI-assisted workflow.

---

# Ключевые конкурентные преимущества NeoRecorder

NeoRecorder должен быть сильнее OBS в следующих точках:

## 1. Простота входа

Пользователь должен запускать студию быстрее и проще, чем в OBS.

## 2. Более понятный UX

Меньше технической свалки, больше осмысленной структуры.

## 3. Меньше разрыва между записью и публикацией

Записал -> получил транскрипт -> получил хайлайты -> экспортировал клипы.

## 4. Умные автоматические решения

Не пользователь ищет проблему, а система подсказывает её сама.

## 5. Более современная архитектура

Без огромного слоя исторических компромиссов и легаси, характерных для старого продукта.

---

# Что должно стать killer-feature

Если выбирать один главный сценарий, который реально может взорвать рынок, то это должен быть следующий пользовательский поток:

1. Пользователь быстро собирает сцену.
2. Записывает видео или проводит эфир.
3. После завершения NeoRecorder автоматически:

   * делает транскрипт;
   * находит лучшие моменты;
   * предлагает 3–5 клипов;
   * создаёт субтитры;
   * предлагает название и описание;
   * даёт экспорт под YouTube / Shorts / TikTok.

Если это будет работать действительно хорошо, это уже будет не просто конкурент OBS, а продукт другого поколения.

---

# Как вести разработку, чтобы не умереть от амбиций

Амбиция должна быть большой.
Реализация — жёстко по приоритетам.

## Правило разработки

Каждая новая фича должна проходить через такой фильтр:

1. Усиливает ли это студийное ядро?
2. Улучшает ли это live-workflow?
3. Снижает ли это трение для пользователя?
4. Приближает ли это интеграцию записи и пост-обработки?
5. Создаёт ли это реальное преимущество над OBS?

Если ответов «да» нет — фича не приоритетна.

# Определение успеха

Проект движется правильно, если через несколько итераций можно честно сказать:

* NeoRecorder стабилен как рабочий инструмент;
